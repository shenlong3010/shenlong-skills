# shenlong-skills

Personal Claude Code toolbox: 47 skills, 18 slash commands, 10 subagents, lifecycle hooks, MCP scaffolds, output styles, instruction templates, and a self-healing execution loop — packaged as a plugin so one install makes everything available in any project.

## Install

**Plugin (preferred):** add this repo as a local plugin — the `.claude-plugin/plugin.json` manifest wires commands, agents, skills, and hooks.

**Symlink fallback:** `./install.sh` links leaf folders into `~/.claude/skills`, `~/.claude/commands`, and `~/.claude/agents`.

**Copilot / other surfaces:** no auto-load; reference the SKILL.md text manually. Hooks and plugin machinery do not transfer.

Every artifact carries two orthogonal frontmatter axes — `flow:` (workflow stage) and `domain:` (topic) — enforced by `tools/validate.py` and rendered as grouped catalogs in `skills/CLAUDE.md`, `commands/CLAUDE.md`, `agents/CLAUDE.md` (generated by `tools/gen-index.py`; skill bodies lazy-load on trigger, only descriptions are front-loaded).

## The ralph loop

Self-healing multi-story implementation loop (concept adapted from [snarktank/ralph](https://github.com/snarktank/ralph)): a thin orchestrator delegates real work to fresh-context role subagents, driven entirely by on-disk state — cross-chat resume is free, nothing lives in the transcript.

```
/ralph plan     stories → runs      (adapter ladder: Jira MCP → GitHub → Bitbucket → local .agents/BACKLOG.md)
/ralph next     one iteration       (read state → drift check → spawn ONE role → write state → stop)
/ralph auto     loop to done        (budgets, PAUSE sentinel, halt markers)
/ralph status   derived view        (zero writes — generated from run dirs, cannot go stale)
/ralph replan   inward open loop    (refetch story, regen affected tasks, [v] untouched, confirmed)
/ralph tickets  outward open loop   (flush discovered-work proposals — only after per-row human confirm)
```

- **Roles:** `ralph-coder` (implements one task to its AC), `ralph-task-inspector` (verifies by running things; literal `VERDICT: PASS|FAIL|ERROR|DRIFT`), `ralph-diagnostician` (retries exhausted; mode `false|propose|rewrite` — propose diagnoses and hands spec changes to a human via HANDOFF.md).
- **State machine:** `[ ]`→`[x]`→`[v]`, off-path `[!]` retry · `[R]` spec-rewritten · `[H]` human-halt (always writes a handoff) · `[D]` drift. FAIL charges the retry budget; ERROR halts; DRIFT charges nothing.
- **Steering:** `LOOP-CONTEXT.md` (`Guidance` human-written, `Learned` capped at 10 bullets) prepended to every spawn; `PAUSE: true` brakes a live auto loop from disk. `LOCK` file arbitrates concurrent sessions.
- **Inputs are ephemeral:** fetch → execute → delete. Backlogs and fetched stories live under gitignored `.agents/`, never committed. Budgets in `.agents/ralph.yml`.
- **Run completion:** `## Summary` metrics in PROGRESS (self-healing is measured, not claimed) + Learned-bullet promotion to evals, Guidance, or skill gotchas.

Templates for all run artifacts: `templates/ralph/`. Adapter capability model: `skills/ralph-plan/references/adapters.md`.

## Evals

`evals/` holds golden cases in the native `claude plugin eval` layout (`prompt.md` + `graders/criteria.md`); `/eval-writer` authors new ones from run traces. Routing cases pin that the right skill lane fires (symbol vs grep, structured parse vs text-grep, ranked hypotheses vs guessing). Runner is early-access gated at time of writing — cases accumulate as the regression backlog and run unchanged once it opens:

```bash
claude plugin eval shenlong-skills --tag routing --runs 1 --max-cost-usd 2
```

## Layout

```
.claude-plugin/   plugin manifest (hooks wired here)
skills/           agent-consumed skills (folder per skill, SKILL.md + optional references/)
commands/         slash commands (flat .md)
agents/           subagents (flat .md)
evals/            golden eval cases (native claude plugin eval layout)
hooks/            lifecycle hook handlers (guard-dangerous, autoformat, notify, cost-logger)
mcp/              MCP configs and scaffolds (env-var keys only — never commit credentials)
styles/           output styles
templates/        CLAUDE.md / AGENTS.md starters, settings presets, templates/ralph/ run scaffolds
tools/            validate.py, skill-lint.py, knowledge-lint.py, gen-index.py, scaffold.py, scan.sh
plan/             the build plans (PLAN-1 … PLAN-5)
```

Instruction hierarchy: root `CLAUDE.md` is global; `skills/`, `agents/`, `commands/`, `tools/` each carry a directory `CLAUDE.md` with local conventions and a generated catalog; `archive/CLAUDE-v2-full.md` is the behavioral master; `AGENTS.md` and `.github/copilot-instructions.md` are global-only surface adapters.

## Authoring

Use the creators — do not hand-roll files:

```
/create-skill <name>     /create-command <name>     /create-agent <name>
/create-tool <name>      /create-hook <name>
```

All five call `tools/scaffold.py <kind> <name> [flow] [domain]`, which writes standard frontmatter and runs `tools/validate.py` immediately. Before any commit: all three linters green (`validate.py`, `skill-lint.py`, `knowledge-lint.py`) and `scan.sh` clean. On Windows, run the Python tools with `PYTHONUTF8=1`.

## Guard

This repo is public. `tools/scan.sh` (wired via `.pre-commit-config.yaml`) scans staged changes for secret patterns and for terms in a local `.wordlist` file. The wordlist is **gitignored and must be authored locally before the first real commit** — an absent or empty wordlist makes that half of the scan pass vacuously. Scan output prints file and line only, never the matched term. Migrated hand-written skills must pass the scan before their first commit.

## Vendoring

Third-party skills are pulled with `tools/vendor_anthropic.sh` (pins a SHA, reminds you to verify the upstream license before committing). Vendored files keep their upstream license; carry a `source:` line in frontmatter and a credits entry below.

**Exception — Anthropic document skills (docx / pdf / pptx / xlsx):** these are `license: Proprietary` (all rights reserved; see each skill's LICENSE.txt) and must never be committed or republished. `tools/fetch-doc-skills.sh` pulls them into `vendor/anthropic/` (gitignored) for local use. The canonical `skills/{docx,xlsx,pptx,pdf}` are committable originals (python-docx / openpyxl / python-pptx / pypdf + reportlab); the vendored proprietary set stays optional for advanced OOXML features (tracked changes, comments, template surgery).
