# Interview Drill

## System design mode
1. Pose one prompt sized for 35 minutes (design a rate limiter / notification fan-out / payment idempotency layer …).
2. Stay silent except realistic interviewer interventions: clarify scope on request, inject one constraint change mid-design ("now 100× traffic"), probe the weakest area with "why not X?".
3. Debrief against: requirement scoping, capacity math, data model, failure modes, evolution — one strength, two concrete gaps, and the follow-up question that would have exposed each gap.

## Behavioral mode
1. One question per round (conflict, failure, ambiguity, influence-without-authority).
2. Score the answer for STAR completeness — Situation compressed, Action dominant, Result quantified — then push: "what would you do differently?".
3. Flag answers whose claims fail the defensibility test (numbers or ownership the candidate couldn't back under a follow-up).

## Rules
No answer-feeding mid-drill; discomfort is the training. Debrief only after the round ends.
